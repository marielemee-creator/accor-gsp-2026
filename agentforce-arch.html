BYOM
AccorGPT ne voit que ce que SF lui envoie explicitement. Le PII masquage se fait dans le Trust Layer avant l'envoi. Aucune donnée CRM ne quitte SF sans être filtrée. Le Responsible AI Committee d'Accor peut auditer chaque prompt envoyé.
⚙️
Enregistrement BYOM : AccorGPT s'enregistre dans AI Gateway → Model Registry de Salesforce. L'endpoint HTTP, les credentials et les paramètres de routing sont configurés dans Agentforce Studio — pas de code. Disponibilité SF BYOM : Q4 2026 / Q1 2027 — à reconfirmer sur roadmap.
⚙️ Notes techniques — pour l'équipe ingénierie
BYOM — enregistrement modèle externe
AccorGPT enregistré via AI Gateway. SF envoie un payload JSON structuré (prompt + contexte filtré). La réponse est retournée au bot/agent dans la console.
Einstein Trust Layer — PII & audit
Masquage PII automatique avant envoi au modèle. Audit trail stocké dans Event Monitoring SF. Aucun prompt ni réponse stocké côté SF sauf si configuré explicitement.
Topics et Actions — configuration sans code
Topics configurés dans Agentforce Builder (UI). Actions = Apex invocables ou Flows existants. Pas de développement ML requis — SF orchestre, AccorGPT génère.
Déploiement par canal
Un même agent Agentforce peut être déployé sur Voice (via Agent Assist), Chat (via Omnichannel bot), Email (via Case Classification) — sans re-développement.
